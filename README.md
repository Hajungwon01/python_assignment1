# 파이썬 과제 1 (python_assignment1)

본 프로젝트는 파이썬의 핵심 기초인 제어문, 자료구조(List, Dictionary), 그리고 모듈화를 실습하고, pickle 모듈을 이용한 데이터의 영구 저장을 구현한 과제입니다.

## 1. 프로젝트 구조
python_assignment1/
 |- main.py             # 프로그램 메인 엔트리 및 메뉴 컨트롤러
 |- loop/               # 반복문 및 리스트 실습 패키지
    |- while_mission.py # 성적 관리 프로그램 모듈
 |- fileio/             # 파일 입출력 및 딕셔너리 실습 패키지
    |- fileio_mission.py# 직원 정보 관리 서비스 모듈

## 2. 모듈별 구현 상세

### [Main] main.py
- menu() 함수를 통해 메인 프롬프트를 반복 실행합니다.
- 사용자 입력(1, 2, 9)에 따라 모듈별 프로세스를 호출하며, 잘못된 번호 입력 시 예외 처리 문구를 출력합니다.

### [Loop] 성적 관리 프로그램 (while_mission.py)
- 함수명: sungjuk_process()
- 데이터 구조: sungjuk_list = [] (중첩 리스트 구조)
- 기능 세부사항:
    - 추가 (info_add): 학생 번호(sno), 이름(sname), 점수(score)를 입력받아 리스트에 추가합니다.
    - 삭제 (info_remove): 리스트의 인덱스(순번)를 입력받아 데이터를 삭제합니다.
        - 방어 코드: 리스트가 비어있을 경우 "현재 등록된 학생 정보가 없습니다." 문구를 출력하고 리턴합니다.
        - 유효성 검사: 잘못된 순번 입력 시 올바른 번호를 입력할 때까지 반복해서 재입력을 요구합니다.
    - 출력 (info_print): while 문을 사용하여 저장된 모든 학생 정보를 인덱스와 함께 한 줄씩 출력합니다.

### [File I/O] 직원 정보 관리 서비스 (fileio_mission.py)
- 함수명: emp_process()
- 데이터 구조: emp_dict = {} (사번을 Key로, 정보 리스트를 Value로 사용)
- 기능 세부사항:
    - 추가 (emp_add): 사번, 이름, 급여, 부서 등 8가지 정보를 입력받아 딕셔너리에 저장합니다.
    - 삭제 (emp_remove): 삭제할 사번을 입력받아 아이템을 제거합니다.
        - 방어 코드: 데이터가 없을 경우 삭제 불가 메시지를 출력합니다.
        - 유효성 검사: 존재하지 않는 사번 입력 시 재입력을 유도합니다.
    - 전체 출력 (emp_print): 딕셔너리의 Key 리스트를 추출하여 while 문으로 전체 직원을 출력합니다.
    - 파일 저장 (emp_file_save): pickle.dump()를 사용하여 사용자가 지정한 파일명(.dat)으로 데이터를 저장합니다.
    - 파일 로드 (emp_file_read): pickle.load()로 파일 데이터를 읽어와 메모리의 emp_dict에 저장하고 출력합니다.

## 3. 주요 특징 (기능 강화)
- 공백 데이터 체크: 삭제나 출력 시 데이터가 0개인 상황을 미리 체크하여 프로그램의 안정성을 높였습니다.
- 무한 루프 활용: 잘못된 데이터 입력 시 프로그램을 종료하지 않고 사용자가 올바른 값을 입력할 때까지 재입력을 받도록 설계되었습니다.
